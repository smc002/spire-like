# 02 战斗系统

## 回合时序

```
战斗开始
  ├─ 触发"战斗开始"效果（遗物、能力卡）
  ├─ 玩家初始手牌 = 抽 5 张
  └─ 进入第 1 回合（玩家先手）

玩家回合
  ├─ 回合开始
  │   ├─ 能量重置为 3（基础值，可被遗物/能力修改）
  │   ├─ 抽至 5 张（不足则从抽牌堆补；抽牌堆空时把弃牌堆洗回）
  │   ├─ 触发"回合开始"效果
  │   └─ 敌人公布本回合意图
  ├─ 行动阶段
  │   ├─ 任意顺序出卡（消耗能量、立即结算）
  │   ├─ 可主动使用药水
  │   └─ 玩家点击"结束回合"
  └─ 回合结束
      ├─ 触发"回合结束"效果（毒、燃烧等）
      ├─ 手牌进弃牌堆（除留存型）
      ├─ 临时状态衰减（护甲清零、易伤/虚弱回合数 -1）
      └─ 切到敌方回合

敌方回合
  ├─ 每个敌人按公布的意图执行（攻击/防御/施法/特殊）
  ├─ 触发对应状态（中毒、燃烧等）
  └─ 每个敌人决定下回合意图，切回玩家

战斗结束
  ├─ 所有敌人 HP ≤ 0 → 胜利结算
  └─ 玩家 HP ≤ 0 → 跑团失败
```

## 卡牌区

| 区域 | 说明 | 上限 |
|------|------|------|
| 抽牌堆（Draw Pile） | 本场战斗剩余卡，玩家不可见顺序 | 无 |
| 手牌（Hand） | 当前可出 | 10 |
| 弃牌堆（Discard Pile） | 已出的卡；抽牌堆空时洗回 | 无 |
| 消耗堆（Exhaust Pile） | 本场战斗永久移除（战斗结束回归牌组） | 无 |

**关键规则**：
- 手牌到达上限后，新抽的卡直接进弃牌堆
- 抽牌堆空时洗弃牌堆回抽牌堆（瞬间，无动画延迟）
- 消耗堆不参与洗牌
- 抽牌堆+弃牌堆都空时，抽牌指令静默失败（不视为错误）

## 能量

- 基础每回合 3 点
- 部分遗物/能力可永久增加（如「+1 能量永久」）
- 未用完的能量回合结束清零，除非有「留存」类效果
- 卡牌花费可为 `0`、`1`、`2`、`3` 或 `X`（X = 当前所有能量）

## 伤害与护甲

```
受击 = max(原始伤害 - 当前护甲, 0)
护甲消耗 = min(原始伤害, 当前护甲)
```

- 护甲在角色回合结束时清零（除非有「护甲保留」buff）
- 多源伤害分别结算（如「3 次攻击各造成 4 伤害」=3 次独立结算）
- 失去 HP 直接绕过护甲（敌方部分技能、自损卡）

## 卡牌类型

| 类型 | 说明 | 出牌后去向 |
|------|------|----------|
| 攻击（Attack） | 对敌人造成伤害 | 弃牌堆 |
| 技能（Skill） | 防御、抽牌、施加状态 | 弃牌堆 |
| 能力（Power） | 战斗内永久效果（如「每回合 +2 护甲」） | 消耗堆（不是真的消耗，是"附加到玩家身上"） |
| 状态（Status） | 通常无益、占手牌位（如「灼伤」） | 弃牌堆或消耗堆 |
| 诅咒（Curse） | 无法主动出（部分例外） | 弃牌堆 |

## 卡牌属性骨架

```
id:           StringName     # 唯一标识，如 "strike"
display_name: String         # 显示名
cost:         int            # -1 表示 X 费
type:         Card.Type      # ATTACK/SKILL/POWER/STATUS/CURSE
rarity:       Card.Rarity    # BASIC/COMMON/UNCOMMON/RARE
target:       Card.Target    # ENEMY/ALL_ENEMIES/SELF/NONE/RANDOM
effects:      Array[Effect]  # 卡牌效果序列（攻击/格挡/抽牌/施加状态…）
upgraded:     bool           # 是否已升级
exhaust:      bool           # 出牌后入消耗堆
ethereal:     bool           # 回合结束若仍在手则消耗
innate:       bool           # 战斗第一回合必抽到
retain:       bool           # 回合结束不入弃牌
```

## 事件总线（实现层关键）

战斗中通过 `BattleEvents` 单例发送以下信号；遗物、状态、能力、特殊卡都订阅它们而不耦合战斗管理器：

| 事件 | 触发时机 | 携带数据 |
|------|---------|---------|
| `battle_started` | 战斗初始化完成 | enemies, deck |
| `turn_started` | 任何角色回合开始 | actor |
| `turn_ended` | 任何角色回合结束 | actor |
| `card_drawn` | 抽到一张卡 | card |
| `card_played` | 卡牌结算前（可被修改/取消） | card, targets, context |
| `card_resolved` | 卡牌结算完 | card |
| `card_discarded` | 卡进弃牌堆 | card, reason |
| `card_exhausted` | 卡进消耗堆 | card, reason |
| `damage_intent` | 伤害即将结算（可被修改） | source, target, amount, type |
| `damage_dealt` | 伤害结算后 | source, target, amount, blocked |
| `block_gained` | 获得护甲 | target, amount |
| `status_applied` | 状态附加 | target, status, stacks |
| `enemy_intent_changed` | 敌人意图变更 | enemy, new_intent |
| `battle_won` / `battle_lost` | 战斗结束 | (无) |

**为什么用事件总线**：力量、易伤、暴怒、各类遗物的效果几乎都是"在某事件发生时插入逻辑"。事件总线让这些行为独立、可测、可热插拔。

## 优先级与拦截

事件总线信号按订阅顺序串行处理；修饰类效果（如易伤、力量）需要在伤害最终结算前完成所有修饰。设计上把 `damage_intent` 设为"可修改"，按以下顺序串接：

```
原始伤害
  → 力量加成（+力量值）
  → 虚弱减成（×0.75，若来源虚弱）
  → 易伤加成（×1.5，若目标易伤）
  → 其他遗物/能力修饰
  → 最终伤害
  → 护甲扣除
  → HP 扣除
```

实现时把这条链做成有序的 `damage_modifier` 数组，每个 modifier 是 `(原值, 上下文) → 新值` 的纯函数，方便加新效果。
